# OpenCode Task Manager Plugin 设计说明

## 概述

OpenCode Task Manager 是一个任务队列插件，用于批量处理文件。它扫描指定目录下的文件，为每个文件创建独立的 Session 执行任务，并自动串行执行直到所有任务完成。

## 核心功能

### 1. 任务队列管理

- **扫描目录**：支持相对路径和绝对路径
- **文件过滤**：按文件后缀匹配（如 `.java`、`.ts`）
- **递归扫描**：支持子目录递归扫描
- **串行执行**：任务一个接一个自动执行

### 2. Session 管理

- **独立 Session**：每个任务创建独立的 Session
- **标题命名**：Session 标题为文件名，便于识别
- **右侧任务栏**：Session 显示在 OpenCode 右侧任务栏

### 3. 结果汇总

- **状态追踪**：成功/失败状态
- **结果摘要**：每个任务的执行结果
- **错误记录**：失败任务的错误信息

## 关键设计决策

### 1. 按项目目录隔离队列

使用 `Map<directory, QueueState>` 存储队列状态，确保不同项目的队列不会互相干扰。

### 2. 单文件插件

所有代码在一个文件中，简化部署和维护。

### 3. 同步执行

使用 `await session.prompt()` 等待每个任务完成，确保串行执行。

### 4. 文件路径处理

- 支持相对路径和绝对路径
- 递归扫描正确计算相对路径
- Session 标题使用文件名
- Prompt 包含文件绝对路径

### 5. 超时设置

Session 完成等待时间设置为 1 小时，适应长时间任务。
